# 2021 CSP-J1 入门级 C++语言试题：答案详解与考点 tag 权重

## 考点 tag 口径

本版采用简洁 tag 体系，对照《全国青少年信息学奥林匹克系列竞赛大纲（2023 年修订版）》中 CSP-J 对应的 `2.1 入门级` 知识范围进行归类。

- 每题可包含多个 tag，权重合计 100%。
- tag 只保留到便于统计和讲题的粒度，例如 `循环`、`循环嵌套`、`数组`、`字符串`、`结构体`、`递归`、`动态规划`。
- 不拆成“定义、使用、嵌套、相关函数”等过细子项；这些内容统一归入对应简洁 tag。

## 一、单项选择题

### 1. 答案：D

**详细解析：** C++、Python、Java 都支持面向对象程序设计；C 语言主要是过程式语言，不属于典型的面向对象程序设计语言。


**考点 tag 与权重：** 程序设计语言 70%；基础知识 30%

### 2. 答案：B

**详细解析：** 图灵奖是计算机科学领域的重要奖项，常被称为“计算机界的诺贝尔奖”。其他选项分别偏向电影、综合学术或新闻文学领域。


**考点 tag 与权重：** 计算机常识 80%；基础知识 20%

### 3. 答案：A

**详细解析：** 计算机底层用电路的高低电平表示数据，因此主流计算机最终都以二进制形式存储和处理数据。


**考点 tag 与权重：** 二进制 70%；计算机基础 30%

### 4. 答案：C

**详细解析：** 要在 $N$ 个数中找最大值，先把第一个数当作当前最大值，再与剩余 $N-1$ 个数逐一比较，最少也需要 $N-1$ 次比较。


**考点 tag 与权重：** 计数原理 45%；算法分析 35%；枚举 20%

### 5. 答案：D

**详细解析：** 若想先弹出 `c`，需要先压入 `a,b,c`；弹出 `c` 后可以压入并弹出 `d`，但此时栈顶是 `b`，不可能先弹出 `a`，所以 `c,d,a,e,b` 不是合法出栈序列。


**考点 tag 与权重：** 栈 75%；模拟 25%

### 6. 答案：D

**详细解析：** 含 $n$ 个顶点的树有 $n-1$ 条边。原图有 $m$ 条边，要保留连通且变成树，需要删去 $m-(n-1)=m-n+1$ 条边。


**考点 tag 与权重：** 图 50%；树 35%；计数原理 15%

### 7. 答案：C

**详细解析：** $101.11_2=1\times2^2+0\times2^1+1\times2^0+1\times2^{-1}+1\times2^{-2}=4+1+0.5+0.25=5.75$。


**考点 tag 与权重：** 进制转换 75%；数与运算 25%

### 8. 答案：A

**详细解析：** 高度为 5 的完全二叉树，前 4 层必须填满，共 $1+2+4+8=15$ 个结点；第 5 层可以从左到右填入 1 到 16 个结点，因此有 16 种不同形态。


**考点 tag 与权重：** 完全二叉树 75%；计数原理 25%

### 9. 答案：B

**详细解析：** 先处理括号得到 `b c +`，再与 `a` 相乘得到 `a b c + *`，最后再与 `d` 相乘，后缀表达式为 `abc+*d*`。


**考点 tag 与权重：** 表达式 45%；栈 35%；模拟 20%

### 10. 答案：B

**详细解析：** 6 个人分成 3 个无编号的两人队，可以先全排列再除去每队内部顺序和队伍之间顺序：$\frac{6!}{(2!)^3\cdot3!}=15$。


**考点 tag 与权重：** 排列组合 75%；计数原理 25%

### 11. 答案：B

**详细解析：** 哈夫曼编码每次合并权值最小的两个结点，是局部最优逐步构造全局最优编码树的过程，本质上属于贪心策略。


**考点 tag 与权重：** 哈夫曼树 60%；贪心 30%；树 10%

### 12. 答案：A

**详细解析：** 可按数字组成情况计数：`1,2,3` 三个不同数字有 $3!=6$ 种；含两个 1 的三位数有 `{1,1,2}` 和 `{1,1,3}`，各 3 种；含两个 2 的三位数有 `{2,2,1}` 和 `{2,2,3}`，各 3 种。总数为 $6+3+3+3+3=18$。


**考点 tag 与权重：** 排列组合 70%；计数原理 20%；枚举 10%

### 13. 答案：C

**详细解析：** `solve(7)=7*solve(5)`，`solve(5)=5*solve(3)`；而 `solve(3)=3*solve(2)=3*2*solve(1)=6`，所以结果为 $7\times5\times6=210$。


**考点 tag 与权重：** 递归 70%；模拟 20%；数与运算 10%

### 14. 答案：B

**详细解析：** 图中边为 `a-b,a-c,b-d,c-d,c-e`。从 `a` 开始深搜时，若先走 `b` 方向，最后可能访问 `e`；若先走 `c` 并优先走 `e`，最后可能访问 `b`。`c`、`d` 不可能成为最后访问点，因此可能作为最后访问点的只有 `b` 和 `e`，共 2 个。


**考点 tag 与权重：** 深度优先搜索 60%；图 25%；模拟 15%

### 15. 答案：B

**详细解析：** 最优方案为：1 和 2 过河用 2，1 返回用 1；4 和 8 过河用 8，2 返回用 2；最后 1 和 2 再过河用 2。总时间 $2+1+8+2+2=15$。


**考点 tag 与权重：** 贪心 45%；模拟 35%；计数原理 20%


## 二、阅读程序

### 16. 答案：×

**详细解析：** 数组 `a` 的大小为 1000，下标范围是 `0~999`。当 `n=1001` 时，读入循环会访问 `a[1000]`，发生下标越界。


**考点 tag 与权重：** 数组 70%；循环 30%

### 17. 答案：×

**详细解析：** `a[i]` 不必全为正整数，例如输入 0 时 `f(0)` 的循环不会执行，`g(0)` 返回 0，不会陷入死循环。


**考点 tag 与权重：** 位运算 55%；循环 30%；分支 15%

### 18. 答案：×

**详细解析：** 对输入 `2,11,9,16,10` 分别计算 `f(x)+g(x)`：`2` 为 `1+2=3`，`11` 为 `3+1=4`，`9` 为 `2+1=3`，`16` 为 `1+16=17`，`10` 为 `2+2=4`，最后一个应为 4，不是 5。


**考点 tag 与权重：** 位运算 55%；模拟 35%；数组 10%

### 19. 答案：√

**详细解析：** `511998` 的二进制中 1 的个数为 16，最低位的 1 所代表的值为 2，所以 `f(511998)+g(511998)=16+2=18`。


**考点 tag 与权重：** 位运算 70%；模拟 20%；函数 10%

### 20. 答案：×

**详细解析：** 如果把 `g` 函数定义移动到 `main` 后面，且没有提前声明，那么 `main` 中调用 `g` 时编译器还不知道该函数，程序不能正常编译。


**考点 tag 与权重：** 函数 70%；编译 30%

### 21. 答案：B

**考点 tag 与权重：** 位运算 75%；数据类型 15%；模拟 10%

### 22. 答案：×

## 三、完善程序

### 34. 答案：D

**详细解析：** 变量 `c` 用来记录已经离开的人数，当离开人数达到 `n-1` 时，只剩最后一个人，循环应停止。因此 ① 为 `c < n - 1`。


**考点 tag 与权重：** 循环 45%；模拟 40%；变量常量 15%

### 35. 答案：C

**详细解析：** `p` 表示当前报数状态，按 0、1 交替变化。报到 1 的人离开，所以当 `p` 为真时淘汰当前人。因此 ② 为 `p`。


**考点 tag 与权重：** 分支 40%；模拟 40%；变量常量 20%

### 36. 答案：C

**详细解析：** 当某人被标记离开后，已经离开的人数要增加 1，所以 ③ 为 `c++`。


**考点 tag 与权重：** 自增自减 45%；模拟 35%；变量常量 20%

### 37. 答案：D

**详细解析：** 只要遇到一个仍在圈中的人，就完成一次有效报数，报数状态需要在 0 和 1 之间切换，所以 ④ 为 `p ^= 1`。


**考点 tag 与权重：** 位运算 35%；模拟 35%；分支 30%

### 38. 答案：B

**详细解析：** 每轮处理完当前位置后，需要移动到下一个人的位置，并在到达末尾后回到开头，所以 ⑤ 为 `i = (i + 1) % n`。


**考点 tag 与权重：** 模运算 45%；循环 30%；模拟 25%

### 39. 答案：B

**详细解析：** 后续去重和二分查找都依赖点按坐标有序排列，因此比较函数应先按 `x` 升序，再按 `y` 升序。① 为 `a.x != b.x ? a.x < b.x : a.y < b.y`。


**考点 tag 与权重：** 结构体 35%；排序 35%；表达式 30%

### 40. 答案：D

**详细解析：** 数组已经按坐标排序，去重时应保留第一个点，以及与上一个已保留点坐标不同的点。因此 ② 为 `t == 0 || !equals(A[i], A[t - 1])`。


**考点 tag 与权重：** 数组 40%；去重 35%；结构体 25%

### 41. 答案：C

**详细解析：** 这里实现的是左闭右闭区间上的二分查找，并在 `cmp(A[mid],p)` 为真时令 `a=mid+1`，否则令 `b=mid`。为了保证收缩区间，应取靠左中点，即 ③ 为 `(a + b) >> 1`。


**考点 tag 与权重：** 二分法 70%；位运算 15%；数组 15%

### 42. 答案：B

**详细解析：** 若 `A[mid]` 的排序位置在目标点 `p` 之前，就应向右半部分查找，所以判断条件为 `cmp(A[mid], p)`。


**考点 tag 与权重：** 二分法 60%；结构体 25%；分支 15%

### 43. 答案：D

**详细解析：** 枚举矩形时可把 `A[i]` 和 `A[j]` 看作左下角与右上角，要求 `A[i].x < A[j].x` 且 `A[i].y < A[j].y`，这样可以避免重复计数，并保证矩形面积大于 0。


**考点 tag 与权重：** 枚举 40%；计算几何 30%；二分法 20%；去重 10%
