# LiteOJ 使用手册

更新时间：2026-06-19

## 登录与账号

访问 LiteOJ 首页后，可注册普通账号或使用管理员账号登录。

默认管理员：

```text
Algor / Wuchuanmin_2003
```

登录后右上角用户名可进入个人主页修改密码。

## 编程题库

导航进入“编程题库”。

题目列表展示：

- 做题状态；
- 题号和标题；
- 标签；
- 难度；
- 通过率。

点击题目进入题面页。管理员在题面页右上角只会看到“编辑”按钮。

## 提交代码

题面页下方是提交区。

语言：

- C++14 默认选中；
- C++11；
- C++17；
- C11；
- Python 3。

O2：

- C++ 默认开启；
- C/Python 不显示 O2 选项。

提交后会进入提交详情页。若仍在评测中，页面会轮询当前提交；离开提交页后轮询停止。

提交时间以 `UTC+8` 显示。

## 查看提交

“提交记录”页展示最近提交。

普通用户只能查看自己的提交；管理员可以查看所有提交。

提交详情包含：

- 状态；
- 分数；
- 最大用时；
- 最大内存；
- 测试点详情；
- 源代码；
- 编译或系统错误信息。

可重测自己的提交；管理员也可以重测全题提交。

## 管理题目

管理员进入“后台管理” -> “管理题目”。

可进行：

- 新增题目；
- 编辑题面；
- 管理测试数据；
- 隐藏/公开；
- 复制；
- 删除；
- 批量公开/隐藏/删除。

### 新增题目

点击“新增题目”后先进入题面编辑。

填写：

- 题号；
- 标题；
- 难度；
- 时间限制；
- 内存限制；
- 是否启用 Special Judge；
- 标签。标签从统一词典中多选，页面展示中文名，系统保存 slug；
- 题面 Markdown；
- 是否公开。

新题默认不公开。创建成功后会自动进入测试数据管理页。

### 编辑题面

编辑页底部有“管理测试点”入口。

题面支持：

- Markdown；
- KaTeX；
- 代码块；
- 图片附件；
- 实时预览。

## 管理测试数据

进入 `/admin/problem/:id/data`。

页面分为：

1. 评测器；
2. 测试点导入；
3. 已有测试点和评分结构。

### 评测器

默认是标准输出评测。

如果题目答案不唯一，启用 Special Judge：

1. 在题面编辑页勾选“启用 Special Judge”；
2. 在测试数据管理页上传 `checker.cpp`；
3. 或把 `checker.cpp` 放进测试数据 zip 根目录。

checker 使用 testlib 风格：

```cpp
#include "testlib.h"

int main(int argc, char* argv[]) {
    registerTestlibCmd(argc, argv);
    // inf: 输入
    // ouf: 用户输出
    // ans: 标准输出
    quitf(_ok, "ok");
}
```

LiteOJ 会按 `input output answer` 顺序传参。

### zip 上传

支持上传 `.zip`。

要求：

- 输入文件扩展名为 `.in`；
- 输出文件扩展名为 `.out` 或 `.ans`；
- 输入输出必须成对；
- 文件名建议只使用字母、数字、下划线、短横线和点；
- `checker.cpp` 可放在 zip 根目录。

上传时可选择覆盖当前测试点。

### 手动录入

切换到“手动录入测试点”后，每个测试点填写：

- 测试输入；
- 标准输出；
- 时间限制；
- 内存限制；
- 分值。

点击“新增一组测试点”继续添加。

### 普通测试点评分

不开启子任务模式时，每个测试点独立计分。

测试点通过得该点分值；失败不得分。

### 子任务评分

开启子任务模式后：

- 所有测试点默认属于 `子任务1`；
- 可添加多个子任务；
- 每个子任务设置一个实际分值；
- 测试点可拖拽到不同子任务；
- 拖拽位置决定测试点编号；
- 组内所有测试点通过时才获得该子任务分值。

可勾选多个测试点后整体拖动。

删除子任务时，该子任务内测试点会回到第一个子任务；只剩一个子任务时不能删除。

## 初赛题库

导航进入“初赛题库”。

支持：

- 按年份、组别、章节、考点筛选；
- 单项选择题；
- 判断题；
- 阅读程序；
- 完善程序。

点击选项后即时判题，显示：

- 正误；
- 正确答案；
- 解析；
- 考点标签。

管理员导入初赛解析时，推荐使用统一标签格式：

```md
**考点与权重：** language-basics: 70%, bitwise: 30%
```

系统只校验固定 slug 表中的标签。导入后页面展示每个 slug 对应的唯一中文名。

完整卷面和解析模板见 `docs/PRELIM_IMPORT_TEMPLATE.md`。

管理员可在后台管理试卷、导入 Markdown 或隐藏题组。

## 初赛模考

导航进入“初赛模考”。

流程：

1. 选择模拟卷；
2. 开始考试；
3. 逐题作答；
4. 提交；
5. 查看报告。

报告会展示总分、每题正误、你的答案和正确答案。

## 数据分析

导航进入“数据分析”。

可按年份和组别筛选。

包含三块：

- 考点出现次数；
- 考点加权分值；
- 考点/年份对照表。

统计以标签 slug 为准，避免“DP、动态规划、动规”被分成多个考点；页面展示统一中文名。

加权规则：

- 一个考点：获得该小题全部分值；
- 两个及以上考点：取权重最高的两个并按比例分配；
- 权重缺失或均为 0：平均分配。

## 后台用户管理

管理员进入“后台管理” -> “管理用户”。

可查看用户列表，并切换普通用户和管理员角色。

## 常见问题

### 管理员密码忘了怎么办？

```bash
docker compose exec app npm run reset-admin -- Algor Wuchuanmin_2003
```

### 上传 zip 后页面很卡怎么办？

测试数据管理页默认不加载测试点正文。若仍然卡顿，检查 zip 中是否包含过大的文件、过多测试点，或浏览器是否正在渲染大量行。

### Special Judge 显示 System Error 怎么办？

优先检查：

- `checker.cpp` 是否能本地用 `g++ -std=c++17` 编译；
- 是否包含 `#include "testlib.h"`；
- 是否调用 `registerTestlibCmd(argc, argv)`；
- 是否读取了正确的 `inf`、`ouf`、`ans`；
- checker 是否超出 `SPJ_TIMEOUT_MS` 或 `SPJ_MEMORY_LIMIT_MB`。

### 题目公开后学生看不到？

检查：

- 题目是否已公开；
- 测试数据是否已保存；
- 页面是否有筛选条件；
- 反向代理缓存是否过期。
