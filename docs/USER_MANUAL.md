# LiteOJ 使用手册

本文面向教师、管理员和学生，说明 LiteOJ 的日常使用方式。

## 1. 登录与账号

打开站点后，右上角可以注册或登录。

默认管理员账号：

```text
admin / admin123
```

正式使用前建议进入后台创建或修改用户，并尽快更改默认密码。当前版本支持普通用户和管理员两种角色。

### 1.1 个人主页与修改密码

登录后点击右上角用户名可进入个人主页。个人主页会显示用户名、角色和注册时间，并提供修改密码功能。修改密码需要输入当前密码、新密码和确认密码；系统会以安全哈希方式保存新密码，不保存明文密码。


## 2. 学生使用流程

### 2.1 编程题训练

1. 进入“编程题库”。
2. 通过关键词、标签、难度筛选题目。
3. 点击题目进入详情页。
4. 阅读题面，选择语言，提交代码。
5. 在“提交记录”中查看结果。
6. 点击某条提交记录可以查看代码、状态和测试点详情。

支持语言：

```text
C11
C++11
C++14
C++17
Python 3
```

C/C++ 默认开启 O2，可在提交时关闭。

### 2.2 CSP 初赛题库练习

1. 进入“初赛题库”。
2. 按年份、组别、题型、知识点或关键词筛选。
3. 点击题目进入练习页。
4. 单项选择题和判断题点击选项后即时判题。
5. 判题后可以查看答案、解析和知识点标签。
6. 阅读程序和完善程序会显示公共代码块，下面按小题作答。

### 2.3 初赛模考

1. 进入“初赛模考”。
2. 选择一套试卷。
3. 点击“开始练习”或“重新练习”。
4. 完成后提交试卷。
5. 查看模考报告，包括得分、答案、解析和知识点。

### 2.4 数据分析

1. 进入“数据分析”。
2. 在年份下拉框中选择一个或多个年份。
3. 选择组别：CSP-J 或 CSP-S。
4. 点击“分析”。
5. 查看：
   - 考点出现次数柱状图；
   - 考点加权分值中空饼图；
   - 考点/年份对照表。

数据分析适合教师判断：某几年试卷中哪些考点出现多、哪些考点分值贡献高。

## 3. 管理员使用流程

管理员登录后，顶部会出现“后台管理”。

### 3.1 新增编程题

路径：

```text
后台管理 → 编程题管理 → 新增题目
```

需要填写：

- 题号：必须是大写字母 + 数字，例如 `P1001`、`ABC12`；
- 标题；
- 难度；
- 时间限制；
- 内存限制；
- 评分方式：OI 按点累计或 ACM 全过得分；
- 输出比较：标准比较、忽略空白、大小写不敏感或浮点误差；
- 标签；
- 是否公开；
- Markdown 题面。

题面建议结构：

````markdown
## 题目描述

...

## 输入格式

...

## 输出格式

...

## 样例 #1

### 输入

```text
...
```

### 输出

```text
...
```

## 数据范围

...
````

题面支持：

- Markdown 标题、列表、表格；
- 代码块；
- 行内公式 `$a+b$`；
- 块级公式 `$$...$$`；
- 图片附件上传。

### 3.2 编辑、复制、隐藏和删除题目

路径：

```text
后台管理 → 编程题管理
```

每道题右侧有：

```text
编辑 | 数据 | 隐藏/公开 | 复制 | 删除
```

说明：

- 编辑：修改题号、标题、题面、限制、标签等；
- 数据：进入测试点管理；
- 隐藏/公开：控制前台是否可见；
- 复制：复制题面、附件和测试点；
- 删除：删除题目及相关测试点记录。

### 3.3 管理测试点

路径：

```text
后台管理 → 编程题管理 → 数据
```

支持两种方式。

#### 手动添加

填写输入、输出、子任务、分值后保存。子任务可以留空；若多个测试点填写同一个子任务名，则该子任务必须全部通过才会获得这一组测试点的分数。

#### zip 批量上传

推荐结构：

```text
1.in
1.out
2.in
2.out
```

也支持：

```text
1.in
1.ans
subtask1/1.in
subtask1/1.out
```

系统会自动配对 `.in` 与 `.out/.ans`。zip 中的目录名会自动作为子任务名，例如 `subtask1/1.in` 会归入 `subtask1`。

#### 评分和比较建议

- OI 按点累计：适合日常训练、分档数据和部分分。
- ACM 全过得分：适合要求所有测试点都通过才算通过的题。
- 标准比较：忽略行尾空白和首尾空白。
- 忽略空白：适合输出表格、序列等只关心 token 的题。
- 大小写不敏感：适合 yes/no 等答案。
- 浮点误差：适合实数输出题，误差默认 `0.000001`。

### 3.4 导入 CSP 初赛试卷

路径：

```text
后台管理 → 初赛题库管理 → 导入试卷
```

需要上传两份 Markdown：

1. 试卷 Markdown；
2. 答案解析 Markdown。

建议格式参考：

```text
seed/prelim/2025-CSP-J1.md
seed/prelim/2025-CSP-J1-solution.md
```

导入前建议先点“解析预览”，确认题目数量、题型和答案解析是否正常。

### 3.5 管理初赛试卷

路径：

```text
后台管理 → 初赛题库管理
```

可以查看已导入试卷，也可以删除试卷。删除试卷会删除该试卷下的整题、小题和相关练习记录。

## 4. 教学使用建议

### 编程题训练

适合日常算法训练、语法练习和小型班级作业。建议每个题目配 2～10 个测试点，正式训练题可以增加边界数据；如果题目有明显的数据分档，可以用子任务分组表达。

### 初赛题库

适合：

- 讲解历年 CSP-J 初赛；
- 按知识点筛题；
- 课后选择题练习；
- 阅读程序和完善程序专项训练。

### 初赛模考

适合阶段性测验。教师可以让学生统一完成某年试卷，再根据模考报告查漏补缺。

### 数据分析

适合备课时使用。建议选择 3～5 年试卷，观察高频考点和高分值考点，决定复习重点。

## 5. 常见问题

### 5.1 提交一直 Waiting

通常是 judge worker 没启动。

解决：

```bash
npm run judge
```

使用一键同机部署时查看：

```bash
./start.sh status
tail -f logs/judge.log
```

如果只启动了 Web 容器，可以重新拉起宿主机 Docker 沙箱 judge：

```bash
./start.sh restart
```

仅本地/可信内网使用容器内 judge profile 时查看：

```bash
docker compose --profile container-judge logs -f judge
```

### 5.2 C++ 提交 CE

检查：

- 代码是否完整；
- 选择的语言版本是否正确；
- 服务器是否安装了 `gcc/g++`；
- 编译信息是否给出具体错误。

### 5.3 上传 zip 后没有测试点

检查 zip 内是否存在成对文件：

```text
xxx.in + xxx.out
xxx.in + xxx.ans
```

文件名要能配对，系统会忽略没有对应输出文件的输入文件。

### 5.4 局域网设备访问不了

需要：

1. 服务监听 `0.0.0.0`；
2. Windows/服务器防火墙放行端口；
3. WSL2 场景可能需要 `netsh interface portproxy`。

### 5.5 本地 HTTP 下 Cookie Secure 警告

`.env` 中保持：

```text
COOKIE_SECURE=auto
```

本地 HTTP 不会强制 Secure，HTTPS 部署时会自动添加。

## 6. 数据备份

重点备份：

```text
data/
```

其中包含：

- SQLite 数据库；
- 编程题测试点；
- 题面附件。

Docker 部署时数据在卷 `liteoj-data` 中。建议定期导出或挂载到宿主机目录。
