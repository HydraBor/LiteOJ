# 初赛题库 Markdown 导入模板

LiteOJ 初赛题库导入需要同时上传两份 Markdown：

- 卷面文件：只放题目、选项、程序和分值。
- 解析文件：只放答案、考点权重和详细解析。

标签必须来自 `seed/tag-schema.json`，系统只识别 slug，不识别中文名、别名或自由文本。

## 卷面模板

````md
# 2025 CCF 非专业级别软件能力认证第一轮

## 一、单项选择题（共 15 题，每题 2 分，共计 30 分；每题有且仅有一个正确选项）

1.（2分）题目正文。（　）

   A. 选项 A　　B. 选项 B　　C. 选项 C　　D. 选项 D

## 二、阅读程序（判断题正确填√，错误填×）

### （1）

```cpp
01  int main() {
02      return 0;
03  }
```

- 判断题

16.（1.5分）判断题正文。（　）

- 单选题

19.（3分）单选题正文。（　）

   A. 选项 A　　B. 选项 B　　C. 选项 C　　D. 选项 D

## 三、完善程序（单选题，每小题 3 分）

### （1）小标题

题组说明。

```cpp
01  int x = ____①____;
```

34.（3分）①处应填（　）

   A. 选项 A　　B. 选项 B　　C. 选项 C　　D. 选项 D
````

## 解析模板

每题使用二级或三级标题均可，格式必须是：

```md
## 1. 答案：A

**考点与权重：** language-basics: 70%, bitwise: 30%

**详细解析：**

解析正文。
```

题目之间不需要写 Markdown 分隔线 `---`。如果解析文件中保留了单独一行 `---`，LiteOJ 导入时会自动忽略，不会显示在题解中。

判断题答案写法：

```md
## 16. 答案：√

**考点与权重：** loop-structure: 80%, enumeration: 20%

**详细解析：**

解析正文。
```

## 标签规则

- 标签项格式：`slug:权重%` 或 `slug: 权重%`。
- 多个标签可用 `,`、`，`、`;`、`；` 分隔。
- 每题权重建议合计为 `100%`。
- `slug` 必须存在于 `seed/tag-schema.json`。
- 不要写中文标签名、反引号、括号说明或别名。

推荐：

```md
**考点与权重：** dynamic-programming: 80%, array-basic: 20%
```

不推荐：

```md
**考点与权重：** `dynamic-programming`（动态规划）80%；数组 20%
```
