# LiteOJ 使用手册

更新时间：2026-06-27

## 登录与账号

访问 LiteOJ 首页后，可注册普通账号或使用管理员账号登录。

默认管理员：

```text
admin / admin123
```

登录后右上角用户名可进入个人主页修改密码。

## 编程题库

导航进入“编程题库”。

题目列表展示：

- 做题状态；
- 题号和标题；
- 标签；
- 难度；
- 通过率。

列表默认每页 20 条。底部分页条可以切换页码，也可以把每页数量调整为 10、20、50 或 100 条；筛选时会保留当前每页数量并回到第一页。

点击题目进入题面页。管理员在题面页右上角只会看到“编辑”按钮。

## 提交代码

题面页下方是提交区。

语言：

- C++14 默认选中；
- C++11；
- C++17；
- C11；
- Python 3。

O2：

- C++ 默认开启；
- C/Python 不显示 O2 选项。

提交后会进入提交详情页。若仍在评测中，页面会轮询当前提交；离开提交页后轮询停止。

提交时间以 `UTC+8` 显示。

## 查看提交

“提交记录”页展示最近提交。

普通用户只能查看自己的提交；管理员可以查看所有提交。

提交记录默认每页 20 条，底部可切换每页 10、20、50 或 100 条。分页由后端按页返回，提交量较大时也不会一次性拉取全部记录。

提交详情包含：

- 状态；
- 分数；
- 最大用时；
- 最大内存；
- 测试点详情；
- 源代码；
- 编译或系统错误信息。

可重测自己的提交；管理员也可以重测全题提交。

## 管理题目

管理员进入“后台管理” -> “管理题目”。

管理题目列表默认每页 20 条，底部同样支持每页 10、20、50 或 100 条。列表操作列会保持紧凑布局，便于在同一行完成编辑、数据、隐藏、复制和删除。

可进行：

- 新增题目；
- 编辑题面；
- 管理测试数据；
- 隐藏/公开；
- 复制；
- 删除；
- 批量公开/隐藏/删除。

### 新增题目

点击“新增题目”后先进入题面编辑。

填写：

- 题号；
- 标题；
- 难度；
- 时间限制；
- 内存限制；
- 是否启用 Special Judge；
- 标签。标签从统一词典中多选，页面展示中文名，系统保存 slug；
- 题面 Markdown；
- 是否公开。

题号支持常规格式 `P1001`、`ABC12`，也支持复赛题格式 `CSPJ25T1`、`CSPS25T4`。后者会被数据分析识别为 2025 年 CSP-J/S 复赛 T1/T4。题目创建后仍可在题面编辑页修改题号，系统会同步测试数据、附件、提交记录和标签关系。

标签选择区会显示“已选标签”，可直接查看当前勾选结果，也可以点击已选标签移除。

新题默认不公开。创建成功后会自动进入测试数据管理页。

### 编辑题面

编辑页底部有“管理测试点”入口。

题面支持：

- Markdown；
- KaTeX；
- 代码块；
- 表格对齐和表格内 `^` 向上合并单元格；
- `:::align{center}` / `:::align{left}` / `:::align{right}` 对齐块；
- `::cute-table{tuack}` 表格样式块；
- 图片和下载附件，上传后插入题面链接；
- 实时预览。

常用示例：

```markdown
:::align{center}
居中的文字、图片或表格
:::

::cute-table{tuack}

| 测试点编号 | $n \leq$ | $m \leq$ | 特殊性质 |
| :--: | :--: | :--: | :--: |
| $1$ | $1$ | $1$ | AB |
| $2, 3$ | ^ | $10$ | 无 |
```

表格中的 `^` 会和上方同列单元格纵向合并。图片语法为 `![说明](附件地址)`，下载附件语法为 `[文件名](附件地址)`。附件保存时会尽量保留上传文件名；同一题目中上传同名附件会覆盖旧文件。

## 管理测试数据

进入 `/admin/problem/:id/data`。

页面分为：

1. 评测器；
2. 测试点导入；
3. 已有测试点和评分结构。

### 评测器

默认是标准输出评测。

如果题目答案不唯一，启用 Special Judge：

1. 在题面编辑页勾选“启用 Special Judge”；
2. 在测试数据管理页上传 `checker.cpp`；
3. 或把 `checker.cpp` 放进测试数据 zip 根目录。

checker 使用 testlib 风格：

```cpp
#include "testlib.h"

int main(int argc, char* argv[]) {
    registerTestlibCmd(argc, argv);
    // inf: 输入
    // ouf: 用户输出
    // ans: 标准输出
    quitf(_ok, "ok");
}
```

LiteOJ 会按 `input output answer` 顺序传参。

### zip 上传

支持上传 `.zip`。

要求：

- 输入文件扩展名为 `.in`；
- 输出文件扩展名为 `.out` 或 `.ans`；
- 输入输出必须成对；
- 文件名建议只使用字母、数字、下划线、短横线和点；
- `checker.cpp` 可放在 zip 根目录。

上传时可选择覆盖当前测试点。

### 手动录入

切换到“手动录入测试点”后，每个测试点填写：

- 测试输入；
- 标准输出；
- 时间限制；
- 内存限制；
- 分值。

点击“新增一组测试点”继续添加。

### 普通测试点评分

不开启子任务模式时，每个测试点独立计分。

测试点通过得该点分值；失败不得分。

### 子任务评分

开启子任务模式后：

- 所有测试点默认属于 `子任务1`；
- 可添加多个子任务；
- 每个子任务设置一个实际分值；
- 测试点可拖拽到不同子任务；
- 拖拽位置决定测试点编号；
- 组内所有测试点通过时才获得该子任务分值。

可勾选多个测试点后整体拖动。

删除子任务时，该子任务内测试点会回到第一个子任务；只剩一个子任务时不能删除。

测试点管理页支持下载全部测试点，也支持勾选多个测试点后下载所选或批量删除。

## 初赛题库

导航进入“初赛题库”。

支持：

- 按年份、组别、章节、考点筛选；
- 单项选择题；
- 判断题；
- 阅读程序；
- 完善程序。

点击选项后即时判题，显示：

- 正误；
- 正确答案；
- 解析；
- 考点标签。

管理员导入初赛解析时，推荐使用统一标签格式：

```md
**考点与权重：** language-basics: 70%, bitwise: 30%
```

系统只校验固定 slug 表中的标签。导入后页面展示每个 slug 对应的唯一中文名。

完整卷面和解析模板见 `docs/PRELIM_IMPORT_TEMPLATE.md`。

管理员可在后台管理试卷、导入 Markdown 或隐藏题组。

初赛题库列表默认每页 20 条，底部分页条支持每页 10、20、50 或 100 条。

后台“初赛题库管理”中，试卷列表和整题概览分别分页，默认均为每页 20 条，避免一次渲染过多题目。

## 初赛模考

导航进入“初赛模考”。

真题卷列表默认每页 20 套，底部可调整每页数量。

流程：

1. 选择真题卷；
2. 开始考试；
3. 逐题作答；
4. 提交；
5. 查看报告。

报告会展示总分、每题正误、你的答案和正确答案。

## 数据分析

导航进入“数据分析”。

筛选栏包含：

- 组别：`CSP-J` / `CSP-S`；
- 场次：`初赛` / `复赛`；
- 年份：根据前面的组别和场次自动从对应题库中列出，可多选。

初赛分析包含：

- 考点出现次数；
- 考点加权分值；
- 考点/年份对照表。

复赛分析会从公开编程题库中读取 `CSPJ25T1`、`CSPS25T4` 这类题号，包含：

- T1、T2、T3、T4 题位画像；
- 难度标签分布；
- 题位/考点热力表；
- 复赛题目明细；
- 考点/年份对照表。

其中“T1-T4 题位画像”和“题位/考点热力表”分别独占一行，便于在宽屏和窄屏下阅读。

复赛分析不计算考点权重，只统计标签出现次数。

统计以标签 slug 为准，避免“DP、动态规划、动规”被分成多个考点；页面展示统一中文名。

加权规则：

- 一个考点：获得该小题全部分值；
- 两个及以上考点：取权重最高的两个并按比例分配；
- 权重缺失或均为 0：平均分配。

## 你好小轻

登录后可从导航进入“你好小轻”。

你好小轻只绑定 LiteOJ 用户账号，不读取题库、提交记录或模考数据。每个用户只能看到自己的会话。

支持：

- 新建会话；
- 查看历史会话；
- 重命名会话；
- 删除会话；
- 发送消息并流式查看回复；
- Markdown 和代码块渲染。

系统只保存会话中的 `user` 和 `assistant` 消息，不做长期记忆、摘要或向量数据库。

小轻默认开启助教模式拦截：

- 明确要求“直接给代码 / AC 代码 / 完整程序”时，系统会直接返回学习引导，不调用上游模型。
- 正常粘贴题面不会被硬拦截；题面会连同助教规则一起发送给上游模型。
- 上游模型回复会经过完整代码审查；如果模型输出完整 `main` 程序、超长代码块或“复制提交”式内容，系统会用“隐藏完整代码”替代对应代码部分，并保留其余解释。
- 代写拦截开启时，页面会显示“用户请求分析中 / 小轻思考中 / 小轻回复审查中”等阶段提示，最终再输出审查后的回复。

这类拦截不能保证模型永远不犯错，但可以把关键风险放在服务端控制，避免单纯依赖提示词。

## 后台用户管理

管理员进入“后台管理” -> “管理用户”。

可查看用户列表、切换普通用户和管理员角色，也可以将指定用户密码重置为 `123456`。

## 后台小轻配置

管理员进入“后台管理” -> “小轻配置”。

可配置：

- 是否启用 AI；
- 服务商，默认讯飞星辰，可切换 DeepSeek；
- BASE URL；
- 默认模型，讯飞星辰默认 Qwen3.6-35B-A3B（`xopqwen36v35b`）；
- 每用户每日最大请求数；
- 最大输入字符数；
- 最大输出 tokens；
- 上下文模式：`none` 或 `recent`；
- 最近上下文消息数，默认 `6`。
- 系统提示词；
- 代写请求拦截开关、直接拒绝模板开关和代码片段最大行数。

讯飞星辰需要在服务器环境变量中配置 `XFYUN_API_KEY`，DeepSeek 需要配置 `DEEPSEEK_API_KEY`。后台页面只显示当前服务商的 key 是否已配置，不显示密钥内容。

## 常见问题

### 管理员密码忘了怎么办？

```bash
docker compose exec app npm run reset-admin -- admin admin123
```

### 上传 zip 后页面很卡怎么办？

测试数据管理页默认不加载测试点正文。若仍然卡顿，检查 zip 中是否包含过大的文件、过多测试点，或浏览器是否正在渲染大量行。

### Special Judge 显示 System Error 怎么办？

优先检查：

- `checker.cpp` 是否能本地用 `g++ -std=c++17` 编译；
- 是否包含 `#include "testlib.h"`；
- 是否调用 `registerTestlibCmd(argc, argv)`；
- 是否读取了正确的 `inf`、`ouf`、`ans`；
- checker 是否超出 `SPJ_TIMEOUT_MS` 或 `SPJ_MEMORY_LIMIT_MB`。

### 题目公开后学生看不到？

检查：

- 题目是否已公开；
- 测试数据是否已保存；
- 页面是否有筛选条件；
- 反向代理缓存是否过期。
